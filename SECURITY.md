# Security Policy

## Supported versions

We release security updates for the current major version. We recommend staying on the latest release.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| older   | :x:                |

## Reporting a vulnerability

If you believe you’ve found a security vulnerability, please report it responsibly:

1. **Do not** open a public GitHub issue for the vulnerability.
2. Email the maintainers (or open a **private** security advisory on GitHub: Repository → Security → Advisories → “Report a vulnerability”) with:
   - A short description of the issue
   - Steps to reproduce
   - Impact (e.g. data exposure, privilege escalation)
   - Any suggested fix or mitigation, if you have one

We will acknowledge your report and work with you to understand and address it. We may ask for more detail. We will keep you updated on the status of the fix and credit you in the advisory/release notes if you wish.

## What to expect

- We aim to respond to security reports within a few days.
- We will work on a fix and coordinate disclosure (e.g. release + advisory) once a patch is ready.
- We follow responsible disclosure: we won’t make the issue public before a fix is available (or we agree on a timeline with you).

Thank you for helping keep Lore and its users safe.
